任天堂台灣遊戲監看網站

Switch / Switch 2 即時資訊與特價追蹤 — 執行計畫 v2（2026-05-29，決策已確認）

1. 需求摘要

打造一個類似「遊戲箱子」的網頁，自動爬取任天堂台灣電商商品資料，提供搜尋、收藏、特價追蹤功能。

已確認的設計決策

  • 資料來源：任天堂台灣官網直接爬（store.nintendo.tw / ec.nintendo.com/TW/zh）
  • 更新頻率：每小時（用排程自動跑爬蟲）
  • 部署方式：GitHub Pages（repo：enix007-create/nintendo-tw）
  • 收藏儲存：localStorage + 匯出/匯入 JSON 按鈕
  • 失敗通知：Email → enix_007@yahoo.com.tw
  • UI 風格：參考遊戲箱子（卡片網格、封面為主視覺、特價徽章明顯）
✅ 所有架構決策已確認 計畫已依照確認結果定版，可進入階段一。

2. 系統架構

[GitHub Actions 排程 (每小時)]
        │ 觸發
        ▼
[Python 爬蟲 scraper.py]
   ├─ 抓 zh-TW 商品列表與詳情
   ├─ 抓 en 對照中英文名稱
   └─ 輸出 games.json (含價格、特價、封面 URL)
        │ commit & push
        ▼
[GitHub Pages 靜態網站]
   ├─ index.html (前端 UI)
   ├─ app.js (搜尋、排序、收藏)
   ├─ games.json (資料)
   └─ Fuse.js (中英文模糊搜尋)
        │ 使用者瀏覽
        ▼
[localStorage 收藏 + 匯出 JSON 按鈕]

3. 抓取欄位規格

欄位來源備註
遊戲中文名稱zh-TW 商品頁標題—
遊戲英文名稱en 對應商品頁標題用商品 ID / SKU 對應
原價商品頁定價欄位NT$ 數值
現價 / 特價商品頁特價欄位無特價時 = 原價
特價百分比計算(1 - 現價/原價) × 100
特價區間商品頁促銷標籤需解析「限時優惠至 YYYY/MM/DD」
遊戲封面圖商品頁主圖 URL直接連任天堂 CDN，不下載
平台分類 / Tag區分 Switch / Switch 2
商品連結頁面 URL點卡片可跳轉

4. 階段任務與時程

階段一：偵察與爬蟲 PoC

預估 0.5–1 天
  • 用瀏覽器 DevTools 檢視 store.nintendo.tw 是否有 JSON API、還是純 SSR HTML
  • 找出 Switch / Switch 2 分類的列表頁 URL 與分頁規則
  • 確認中英文版的對照方式（商品 ID、URL slug）
  • 觀察反爬蟲機制（User-Agent、Cookie、速率限制）
  • 產出：一份「資料來源筆記.md」+ 抓 10 個商品成功的 PoC 腳本

階段二：完整爬蟲

預估 1–2 天
  • Python + requests + BeautifulSoup（若需 JS 渲染則用 playwright）
  • 處理分頁、節流、重試、錯誤紀錄
  • 輸出 games.json，含上表所有欄位
  • 差異比對：與上次資料比對，紀錄新上架 / 新特價
  • 單元測試：解析函式對固定 HTML 樣本能穩定輸出

階段三：前端網頁

預估 1–2 天
  • 單頁 HTML + Vanilla JS（不引入大型框架以利靜態部署）
  • 卡片式商品清單：封面、中英文名稱、價格（原價刪除線）、特價百分比徽章、特價到期日
  • 搜尋框：用 Fuse.js 同時對中文名與英文名做模糊搜尋
  • 篩選列：只看特價 / 平台（Switch / Switch 2）/ 價格區間
  • 排序：折扣 % 由高到低、價格、特價結束日
  • 收藏功能：心形按鈕 → localStorage；右上「匯出 JSON / 匯入 JSON」按鈕
  • RWD 響應式設計

階段四：部署與自動化

預估 0.5 天
  • 建立 GitHub repo（public 或 private + GitHub Pages Pro）
  • GitHub Actions workflow：cron: '0 * * * *'（每小時），跑爬蟲 → commit games.json → 觸發 Pages 部署
  • 失敗通知：Actions 失敗時寄信或發 Discord webhook
  • 網址：https://<your-user>.github.io/nintendo-tw/

階段五：驗證與優化

預估 0.5 天
  • 抽 10 個商品，人工核對網站 vs 任天堂官網售價、特價是否一致
  • 檢查商品數量是否合理（Switch 數千款）
  • 連續觀察 24 小時排程，確認穩定
  • 效能：JSON 過大時改為分平台拆檔，或前端 lazy render

6. 風險與對策

風險對策
任天堂改版破壞解析器解析失敗時保留上次 JSON、發失敗通知；解析邏輯模組化便於修補
被反爬蟲擋下合理 User-Agent、加延遲、每小時頻率不算激進；必要時降為每 3 小時
商品數過多導致 JSON 太大拆成 switch.json / switch2.json；前端分批載入
GitHub Actions 配額用盡免費 public repo 無上限；private repo 評估改用 Cloudflare Workers Cron
法律 / robots.txt先查 robots.txt，僅作個人非商業用途，不轉售資料
