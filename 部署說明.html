部署與設定說明

任天堂台灣遊戲監看 — MVP 部署到 GitHub Pages

1. 專案結構

nintendo-tw/
├── .github/workflows/hourly-scrape.yml   GitHub Actions 排程
├── scraper/
│   ├── scrape.py                          主爬蟲
│   ├── bilingual_dict.py                  中英文系列對照
│   └── requirements.txt                   Python 套件
├── web/
│   ├── index.html                         前端網頁
│   └── games.json                         即時資料
├── .gitignore
└── 部署說明.html（本檔）

2. 部署步驟（按順序做完）

1建立 GitHub repo

到 github.com/new，用 enix007-create 帳號建立名為 nintendo-tw 的 public repo（public 才有免費 GitHub Pages 與 Actions 額度）。

2把本機程式碼推上去

在 C:\Users\user\Desktop\Claude-workspace\projects\nintendo-tw 目錄開 Git Bash 或 PowerShell：

git init
git add .
git commit -m "init: MVP"
git branch -M main
git remote add origin https://github.com/enix007-create/nintendo-tw.git
git push -u origin main
3啟用 GitHub Pages

到 repo Settings → Pages →

  • Source 選 GitHub Actions（不是 Branch）
  • 第一次部署成功後，會有網址 https://enix007-create.github.io/nintendo-tw/
4設定 Email 通知 Secrets

到 repo Settings → Secrets and variables → Actions → New repository secret，加入兩個：

Secret 名稱內容
MAIL_USERNAME你的 Gmail 地址（例如 xxx@gmail.com）
MAIL_PASSWORDGmail 應用程式密碼（不是登入密碼！見下方）
Gmail 應用程式密碼怎麼取得：到 myaccount.google.com/apppasswords，建立一組 16 碼密碼，貼到 MAIL_PASSWORD。需先開啟兩階段驗證。

失敗時會寄信到你設定的 enix_007@yahoo.com.tw。如果不想用 Gmail 寄，可以把 workflow 中的「Notify on failure」這段拿掉。

5觸發第一次跑

到 Actions 分頁 → 選 "Hourly scrape" → Run workflow（手動觸發一次）。第一次跑會花 2~4 分鐘（裝 Playwright + Chromium）。

之後每小時的第 5 分鐘會自動跑，更新 web/games.json 並重新部署 Pages。

3. 本機開發與測試

跑爬蟲

cd nintendo-tw\scraper
pip install -r requirements.txt
python -m playwright install chromium
python scrape.py

輸出會寫到 ..\web\games.json。

預覽網頁

有兩種方式：

  1. 直接點開 web\index.html（多數瀏覽器允許讀同資料夾的 games.json）
  2. 啟動本機 server： 
    cd nintendo-tw\web
python -m http.server 8000
    然後開 http://localhost:8000

4. MVP 限制與下一步

目前已知限制
  • 遊戲總數約 70 款：只抓任天堂台灣官網兩個目錄頁的 NSUID。實際應有上千款，下一階段需要找「全列表」資料源（可能要直接列舉 NSUID 範圍 + Price API 過濾）
  • 英文搜尋仰賴系列字典：bilingual_dict.py 內建 60+ 組系列名對照，可隨時擴充
  • 盒裝售價未顯示：目前只顯示 Price API 的下載版價（與 eShop 實際一致）

5. 維護重點

  • 新增中英對照：編輯 scraper/bilingual_dict.py，依現有格式新增系列
  • 調整 UI 風格：編輯 web/index.html 內的 CSS 變數（:root）
  • 暫停 Actions：到 Actions → Hourly scrape → ⋯ → Disable workflow
  • 查爬蟲記錄：Actions 分頁可看每次 run 的完整 log
