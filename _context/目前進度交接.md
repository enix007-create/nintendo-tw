## v2 架構（重點）

### 四階段獨立執行 + 持久化 cache

```
scrape.py --phase=catalog    # Playwright 渲染 Switch + Switch 2 目錄頁
scrape.py --phase=scan       # Price API NSUID 範圍掃描（50000–135000）
scrape.py --phase=tw_search  # ★ TW 官方搜尋端點 fan-out（267 keywords，TW 權威來源）
scrape.py --phase=names      # 缺名稱者去 HK store / ec.nintendo.com 補抓（fallback）
scrape.py --phase=build      # Price API 重打、合併、輸出 web/games.json

# 包裝：
scrape.py --phase=refresh    # catalog + tw_search + names(limit=300) + build。hourly
scrape.py --phase=bootstrap  # catalog + scan + tw_search + names(limit=500) + build。daily
```

### Cache 持久化在 `scraper/_cache/`

- `games_meta.json`：核心 cache，存所有已知 NSUID 的 `{platform, name, cover_url, ...}`。每次 commit 進 repo，跨 run 累積
- `scan_nsuids.json`：scan phase 的 raw prices（build phase 拿來當 Price API 被擋時的 fallback）
- `catalog.json`：catalog phase 摘要

### 重要技術細節

1. **Akamai 反爬**：`api.ec.nintendo.com` 對高並行/高頻率超敏感。設定 `SCAN_CONCURRENCY=4`、`SCAN_DELAY=0.25s`、`SCAN_MAX_RETRY=2`、bootstrap 跑完 scan 後 sleep 30s 退火再跑 names
2. **TW 跟 HK NSUID 不共用**（2026-06-01 驗證）：同一款 FF XII THE ZODIAC AGE，TW=`70010000103647`、HK=`70010000016505`，且 cross-region Price API 都回 `not_found`。所以 HK store fallback 只對「兩地剛好共用 NSUID」的款有效；其他款必須靠 TW 自有來源（catalog / tw_search）拿名字
3. **HK 202 retry 沒用**（2026-06-01 驗證）：實測 `202_retry_hit=0/300`，所有 202 都不是 async 生成中，純粹是 HK 沒有那個 NSUID。retry 已從 `_fetch_hk_one` 移除
4. **TW search 上限**：`nintendo.com/tw/search/software?k=<keyword>` 每查詢回上限 ~24 筆，且 `page=2` 無效（會回 page=1 同樣結果）。所以必須用 keyword fan-out 才能擴大覆蓋面
5. **Build fallback**：因為 Price API 在 build 階段常被 403，build 改成先 try Price API refresh，失敗者用 scan cache 的 prices 補上。**這就是為什麼即使 refresh 0 筆，最終還能輸出 248 款**
6. **Push race**：bot commit 跟外部 push 衝突時，workflow 用 `git fetch + reset --hard origin/main + 把產出貼回` 而非 rebase，保證 fast-forward

## CI Workflow 行為

`.github/workflows/hourly-scrape.yml`：

- `cron: 5 * * * *`：每小時的第 5 分鐘觸發
- 條件分流：
  - 整點 03:00 UTC（=台灣 11:00 AM）→ 跑 `--phase=bootstrap`（完整掃描）
  - `workflow_dispatch` 手動觸發 → 跑 bootstrap
  - 其他時間 → 跑 `--phase=refresh`
- `push:` 也會觸發（路徑：`scraper/**` 或 `.github/workflows/**`）→ refresh
- timeout 25 分鐘
- Commit 階段失敗也跑（`if: always()`）— 保住部分 cache
- Push 衝突時 `git reset --hard origin/main + 把產出貼回` 重試 4 次
- 失敗會 email 到 `enix_007@yahoo.com.tw`
